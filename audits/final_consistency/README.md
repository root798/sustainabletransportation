# final_consistency

End-of-run alignment audit, blockers register, and self-evaluation. This folder is written once at the close of the autonomous multi-stage run; any subsequent stage should add a new step folder instead of editing files here.

## Files

| file | purpose |
| --- | --- |
| `FINAL_ALIGNMENT_AUDIT.md` | Repo-wide consistency check across support files, figures, dashboards, backends, scenario templates, shock outputs, and legacy archives. |
| `FINAL_BLOCKERS_AND_RISKS.md` | Register of blockers (need external input), risks (known but not blocking), and polish items. |
| `SELF_EVALUATION.md` | Stage-5 self-evaluation and final checklist. |

## Companion files outside this folder

- `reports/summaries/FINAL_PROJECT_STATUS.md` — final project status (revision-readiness summary).
- `REPORTS_INDEX.md` at repo root — updated in Stage 5 to list this folder.
