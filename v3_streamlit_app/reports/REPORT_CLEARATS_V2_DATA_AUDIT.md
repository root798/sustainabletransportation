## Section B: Data Quality Assessment

### B1. Quantile CSV Validation

**Checked:** Column completeness (all p05/p50/p95 triplets present), monotone year index,
non-negative values, no NaN in primary metric columns.

**Findings:**
- All primary quantile CSVs pass structural validation.
- Year index is monotonically increasing (2024–2092) in all files.
- All energy (kWh) and emission (kg CO2) columns are non-negative.
- Fleet count columns (Total CAV, Total STI) start near zero in 2024 and grow monotonically
  under baseline scenario — consistent with model design.
- Minor: Some early-year values have very wide p05–p95 spread relative to p50 for fleet
  counts, reflecting the large L1 uncertainty on initial fleet data.

**Column naming issue (confirmed):**
- `ATS Total Power (kWh)` and all `*Power (kWh)` columns use incorrect unit label.
- `kWh` denotes energy per year (annual consumption), not instantaneous power.
- Dashboard applies `ENERGY_LABEL_MAP` to correct all display labels.
- Source CSV files should be updated in a future pipeline version.

- California baseline / aggressive / conservative CSVs all have identical structure and year range.

- L3 lognormal distributions use σ parameter (not CV) — consistent with notebook code.
- L1/L2 lognormal distributions use CV parameter — verified against notebook.
- No parameter has SD = 0 or invalid distribution bounds.

## Section D: Limitations and Caveats

### D1. Scientific Limitations

1. **Utility phase only:** Manufacturing, logistics, and end-of-life emissions are not modeled.
   This means CLEAR-ATS underestimates total lifecycle carbon by an unknown amount — potentially
   50–150% more when including manufacturing for battery-intensive EVs.

2. **Fixed model structure:** The adoption curve is logistic growth; retirement is a fixed
   12-year cycle. Alternative functional forms are not explored.

3. **No propulsion energy:** CLEAR-ATS models only the *cyber layer* overhead (sensing, computing,
   communication). Vehicle propulsion (the largest energy consumer) is out of scope.

4. **No spatial variation:** Each region is modeled as homogeneous. Urban vs. rural deployment
   differences within California are not captured.

5. **L3 uncertainty dominates late-year results:** For years 2060–2092, the p95/p05 spread
   exceeds 100% of median in some metrics. These late-year projections have very low precision.

### D2. Data Source Limitations

1. **Consumption rate data (L2):** Per-subsystem energy rates sourced from 2019–2023 hardware
   benchmarks. AV hardware is evolving rapidly; 2030+ rates are highly speculative.

2. **Initial fleet data (L1):** California fleet size from DMV data (~2024). EV fraction
   from California Energy Commission reports.

3. **Grid emission factors (L1):** e_clean = 0.03 kg CO2/kWh, e_fossil = 0.5 kg CO2/kWh.
   These are region-averaged. Marginal emission factors (used in some LCA studies) differ.

4. **ICE power factor (1.6×):** Derived from alternator efficiency assumptions. No empirical
   fleet measurement available for CAV-specific ICE power overhead.

### D3. Dashboard-Specific Limitations

1. **No real-time data:** Dashboard shows pre-computed simulation results, not live data.
2. **No model re-run capability:** Users cannot change parameters and re-run from the dashboard.
3. **Missing scenarios for OH/US:** Only baseline available for Ohio and U.S. Average.
4. **Figures not publication-ready:** Interactive Plotly charts are for exploration;
   publication figures should use the pre-generated PDF outputs in results_notebook/.
